# Схема исходных данных (av-ru.jsonl / ru-av.jsonl)

Источники: `https://sources.avar.me/data/av-ru.jsonl` и `…/ru-av.jsonl`.
Формат JSONL, одна статья на строку. Ниже — реальная схема (изучена по данным)
и подтверждённая семантика полей, включая редкие.

Объёмы на 2026-06: **av-ru 22 855**, **ru-av 38 874** записей.

## Пример записи

```json
{
  "word": "аб", "stress": 1, "stem": "а", "homonym": 2,
  "forms": ["аб", "алъ", "алъул", "ал"],
  "pos": "местоимение", "form": "именительный",
  "gender_forms": {},
  "senses": [
    {
      "text": "это, этот, эта",
      "precomment": "…",
      "comment": "только",
      "labels": ["местоимение", "указательное местоимение"],
      "examples": [{"av": "аб дир къалам буго", "ru": "это мой карандаш",
                    "labels": [], "comment": ""}],
      "masdarfrom": "…", "refwordnum": "1"
    }
  ],
  "see_also": [{"target": "гьаб", "kind": "see"}]
}
```

## Поля верхнего уровня

| Поле | Смысл |
|---|---|
| `word` | заголовок статьи (единственное обязательное поле) |
| `stress` | позиция ударения (число) |
| `stem` | основа |
| `homonym` | номер омонима (одно слово встречается несколько раз; в av-ru ~506 повторов) |
| `forms` | словоформы |
| `pos` | часть речи (существительное, глагол, …; в ru-av обычно пусто) |
| `form` | грамматическая форма заголовка (именительный, масдар, …) |
| `gender_forms` | формы по родам/классам |
| `labels` | пометки на уровне статьи |
| `exclamation` | форма-восклицание заголовка (междометия): `бу` → "бу!" |
| `precomment` | ремарка-префикс ко всей статье (напр. "с назализацией") |
| `senses` | массив значений (см. ниже) |
| `see_also` | перекрёстные ссылки |

## Значение (`senses[]`)

| Поле | Смысл |
|---|---|
| `text` | перевод/толкование (для av-ru — русский; для ru-av — аварский) |
| `precomment` | ремарка **перед** переводом (напр. "выражает испуг, боль") |
| `comment` | ремарка **после** перевода |
| `labels` | пометки значения (часть речи, стиль, "пословица" и т.п.) |
| `examples` | примеры: `{av, ru, labels?, comment?}` |
| `refwordnum` | номер значения/омонима у цели ссылки (когда у цели несколько) |
| грамм. связи | см. ниже |

## Особенности данных

- **Пустые `senses`** (836 в av-ru, 3449 в ru-av) и **senses без `text`**
  (6230 в av-ru) — это легитимные грамматические заглушки-формы, ссылающиеся
  на основную статью. **Не отбраковывать** как «пустые».

## Грамматические связи (значение целевого слова) — резолвятся в `target_id`

`masdarfrom` (масдар от), `genitivefrom` (род. п. от), `pluralfor` (мн. ч. от),
`masdarforceto` (масдар понудит. от), `forceto` (понудит. форма от),
`participlefrom` (причастие от), `deverbfrom` (деепричастие от),
`dativefrom`, `ergativefrom`, `locativefrom`, `ablativefrom`, `casefrom`.

Пример: `алхинабизе` → `forceto: "алхине"` — понудительная форма от «алхине».
`барщараб` → `participlefrom: "барщизе"` — причастие от «барщизе».

## `see_also[]`

`{ "target": "<слово>", "kind": "see" | "from" }` — связанные статьи.
В пайплайне резолвится в `{kind, target_word, target_id}`.

## `link_helper`

Вспомогательная подсказка к ссылке — нормативная/русская форма цели.
Пример: `миндалалъул` → `genitivefrom: "миндал"`, `link_helper: {"genitivefrom": "миндаль"}`
(«миндаль» — русская форма цели, «миндал» — как в аварском тексте). Хранится как есть.

## Как это попадает в SQLite (build_dictionary.py)

- Вся статья (с резолвленными ссылками) — в `entries.data` (JSON).
- Поисковый индекс `search(lang, term, entry_id, weight)`:
  - `weight 0` — заголовок, `1` — формы, `2` — текст значения, (`3` — примеры, выкл. по умолчанию).
  - `lang` = `av` | `ru` — письменность терма; покрывает **оба** словаря
    (поиск по аварскому находит и av-ru заголовки, и аварские переводы из ru-av).
- ID: `dictprefix-sha1(dict+word+homonym+тексты)` с дизамбигуатором при коллизии.
